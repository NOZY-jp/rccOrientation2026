# Detonator 開発プラン全体目次

> 10ドメイン構成の機能分割プラン。
> 各ドメインは独立したプランファイルとして `.sisyphus/plans/` に配置される。
> **各ドメイン完了後に人間が確認してから次のドメインに進む。**

---

## 開発アプローチ: デュアルホスト権威コア方式

ゲームロジックを**純粋なTypeScriptモジュール**として構築し、最初はブラウザで直接動かす。
ColyseusサーバーはD8で**トランスポートアダプタ**として追加する。ロジックの移行は一切発生しない。

## 各ドメインの実行手順（共通）

> 各ドメインのプランファイルに詳細なタスク・QAが記述されていますが、
> 以下は全ドメイン共通の実行チェック手順です。

### 開始前チェック
- [x] 前提ドメインが完了していること（依存チェーン参照）← 00-overviewは前提なし
- [x] `bun run test` → 全テストPASS
- [x] `bun run typecheck` → エラーなし

### ドメイン実行中
- [ ] 各タスクのステップ1（依存インストール）→ `bun add` で必要なパッケージのみ導入
- [ ] 各タスクのステップ2（テスト先書き）→ `bun run test` で RED を確認
- [ ] 各タスクのステップ5（テスト通過）→ `bun run test` で GREEN を確認
- [ ] 各タスクのステップ6（目視確認）→ ブラウザで動作確認し `.sisyphus/evidence/` にスクリーンショット保存

### ドメイン完了チェック
- [ ] `bun run test` → 全テストPASS（D1〜当ドメインまでの累積）
- [ ] `bun run typecheck` → エラーなし
- [ ] 人間がブラウザで動作確認済み
- [ ] コミット作成済み

### 最終検証（全ドメイン完了後）
- [ ] `bun run test` → 全テストPASS
- [ ] `bun run build` → ビルド成功
- [ ] 10フロアクリアの完全プレイテストを人間が実施
