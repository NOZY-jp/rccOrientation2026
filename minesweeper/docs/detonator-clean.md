# ゲームデザイン文書

## エグゼクティブサマリー

本作は、大学新歓イベント向けに開発される協力型ローグライトマインスイーパーパーティーゲームである。従来のマインスイーパーに「荒野システム」という独自の爆発連鎖機構を組み合わせ、時間経過とともに侵食される領域を仲間と守り抜く10〜15分のセッションベースの体験を提供する。

最大の特徴は、すべてのパラメータを数式駆動で設計することで、スキルやアイテムによるゲームルールの動的変更を可能にし、少人数短期開発でも拡張性の高い基盤を構築できる点にある。プレイヤーは各自のデバイスでブラウザアクセスし、リアルタイムで協力しながら、レベルアップによるスキル獲得と買い物を繰り返し、最終的に15分間の生存を目指す。

## ゲームの流れ

プレイヤーは最大10人以上でゲームに参加し、複数のスポーン地点に分散配置される。マインスイーパーのルールに従ってセルを開いていくが、時間経過とともに盤面の未開封セルが地雷エリアに侵食されていく。地雷を踏むと爆発が発生し、周囲8マスが「荒野」化して次の侵食で確実に地雷原となる。侵食インターバルは徐々に短くなり、安全領域が狭まる中で、プレイヤーはレベルアップでスキルを獲得し、Moneyでアイテムを購入しながら生存を目指す。全員が死亡すれば敗北、15分間耐え抜けば勝利となる。

## ゲームシステム

### マインスイーパー基本ルール

本作は古典的マインスイーパーをベースとし、以下のルールを採用する。

- プレイヤーは盤面のセルを開く
- 数字は周囲8マスの地雷数を示す
- 地雷を踏むと爆発が発生
- 和音機能 (chord) は常に有効 (数字の周囲がすべてフラグ済みなら一括オープン)
- 誤フラグペナルティなし

### 荒野システム — コアディファレンシエーター

本作の最大の特徴は、地雷爆発によって生成される「荒野 (wasteland)」と、その連鎖反応である。

**荒野の発生**

プレイヤーが地雷セルを踏むと爆発が発生し、半径1セル（周囲8マス）がすべて荒野化する。爆発範囲内に別の地雷が含まれていた場合、その地雷も起爆し、連鎖反応が発生する。連鎖速度は1セルあたり1/8秒であり、プレイヤーは反射神経で回避可能である。爆発範囲内のプレイヤーは即座に死亡する。

**荒野の性質**

荒野化したセルは、次回の侵食インターバルで必ず地雷原エリアに変換される。これは通常の侵食とは異なり、100%確実に地雷原化する特性を持つ。さらに、荒野は地雷エリアのフロンティアとしてカウントされ、地雷エリアの拡張は荒野の境界からも開始される。

**侵食への影響**

侵食システムは通常、地雷エリアのフロンティアに隣接する未開封セルをBFS（幅優先探索）で拡張していくが、荒野が存在する場合、その境界も拡張起点となる。侵食半径は通常1セルだが、必要セル数が不足する場合はそれを超えて幅優先で拡張される。各侵食フェーズ後、隣接するすべての数字が再計算され、プレイヤーは常に正確な情報を得られる。

```
[爆発連鎖フロー]
プレイヤーが地雷を踏む
        ↓
    半径1セル爆発
        ↓
    荒野化 (8セル)
        ↓
  爆発範囲内に地雷あり？
    Yes → 1/8秒後に連鎖起爆
    No  → 終了
        ↓
  プレイヤーは逃走可能
        ↓
   次回侵食で荒野は地雷原化確定
```

### 侵食システム

時間経過とともに、未開封セルが地雷エリアに変換される。

**侵食トリガー**

侵食はインターバルごとに発生する。インターバルは数式ベースで決定され、時間経過とともに短縮される。侵食セル数もインターバルごとに数式で増加する。侵食対象セルには3〜5秒の警告時間が設けられ、視覚的予告（色変化 + アニメーション）が表示される。

**侵食メカニクス**

1. フロンティア隣接の未開封セルに警告マーカー表示
2. 警告後、対象セルが地雷原に変換（地雷配置率は数式決定）
3. 荒野セルは100%地雷原化
4. 荒野エッジからBFS拡張
5. すべての数字を再計算

**自然なダウンタイム**

長いインターバルは計画・買い物タイムとして機能する。これは数式で管理され、スキルやアイテムで制御可能である。

### 勝利/敗北条件

**勝利**: 15分間生存 → 強制ドラマチックイベント → "Wave Completed!" 表示

**敗北**: 全プレイヤー同時死亡（安全領域がゼロになり侵食で全滅する自然な流れ）

### 死亡とリスポーン

**死亡時**

爆発に巻き込まれる、または地雷原の復活に巻き込まれると死亡する。死亡時、所持アイテムはすべてロストし、リスポーンタイマーが開始される（数式ベース、定数でない）。

**死亡中の状態（LoL方式）**

観戦モード: 他プレイヤーの様子を見られる

ショッピング可能: 死亡中でも買い物ができる

**リスポーンタイマー短縮**

スキルやアイテムでリスポーン時間を削減可能。

## 盤面とプレイヤー

### 盤面構成

ボード規模はかなり大きく、プレイヤー数に応じてスケールする。盤面には複数のスポーン地点が存在し、開いた安全セルが領域として機能する。侵食はフロンティアエッジを狙って進行する。

### プレイヤーの移動と衝突

プレイヤーは1セルの当たり判定を持つ。同一セルに複数プレイヤーは同時占有不可である。実質的に同一セル入力競合は発生しないが、対策は必要である。

### スポーンと合流

スポーン地点は複数存在し、1グループ最大4人まで配置される。プレイヤー数に応じた配分は以下の通り:

- 5人 = 2 + 3
- 6人 = 3 + 3
- 7人 = 3 + 4
- 9人 = 3 + 3 + 3
- 10人 = 3 + 3 + 4

プレイヤーは初期配置を選択できない（システム決定）。合流戦略または分散戦略を自由に選択できる。

## 進行システム

### EXP/Moneyと報酬

本作は通貨 + レベルアップ時ドラフト形式のローグライト進行を実装する。

| 報酬 | 獲得方法 | 用途 | 特性 |
|------|----------|------|------|
| EXP (経験値) | 行動報酬 | レベルアップ | 累積型 |
| Money (Gold) | レベルアップ選択肢 / 行動報酬 | アイテム購入 | 安定資産 |

### レベルアップドラフト

1. EXP蓄積 → レベルアップ
2. ランダムな3つのスキル/アイテムを提示
3. プレイヤーが1つ選択
4. スキル + アイテムスロットが満杯の場合、選択肢はMoneyまたは回復に変化

### ショップ

- Moneyで購入
- いつでも購入可能（インターバル中推奨）
- アイテムは高価
- 緊急用途（早期蘇生など）

### キャラクター選択

プレイヤーは初期にキャラクター選択を行う。キャラクターはガチャテーブルとバフ/デバフに影響する。MVP時に実装はなく、将来的に拡張可能な設計として残される。

### メタプログレッション

- アカウント: ユーザー名 = アカウント（重複不可、セキュリティ考慮なし）
- 永続データ: アカウントレベルで保存
- 1日通しての進行: イベント期間中、アカウント成長が継続
- リトライインセンティブ: メタ進行により再挑戦が有意義に

MVP時に実装はなく、将来的に拡張可能な設計として残される。

## マルチプレイヤー

### 部屋システム

- プレイヤー数: 可変スケーリング（1〜10+人）
- リアルタイム同時操作
- 途中参加: いつでも完全サポート

### リソース管理

**完全個人制**: EXP、Money、アイテム、スキルはすべて個人所有である。他プレイヤーとのリソース共有はなく、協力は行動とタイミングで表現される。

### 協力のあり方

プレイヤーは各自のリソースを管理しながら、盤面の開拓やタイミングの調整を通じて協力する。合流して集中的に安全領域を拡大するか、分散して複数のフロンティアを維持するかは、プレイヤーの判断に委ねられる。

## 技術アーキテクチャ

### 技術スタック

| レイヤー | 技術 |
|----------|------|
| フロントエンド | React, TypeScript, Vite |
| バックエンド | TypeScript (Colyseus) |
| 状態管理 | Zustand |
| レンダリング | Canvas/WebGL (Phaser 3候補) |
| データベース | SQLite3 (メタ進行、ハイスコア) |
| 認証 | ゲストユーザー名のみ (ユーザー名 = アカウント) |
| アーキテクチャ | Authoritative Server |

### システム構成

```
[クライアント (Browser)]
    ↕ WebSocket
[Colyseus Server]
    ↓
[Game State Manager]
    ├─ Board State (侵食、荒野、地雷)
    ├─ Player State (位置、HP、スキル、アイテム)
    ├─ Invasion Timer (数式駆動)
    └─ Event Manager (ランダムイベント)
    ↓
[SQLite3 DB]
    ├─ Account Meta Progression
    └─ High Scores
```

### 数式駆動設計

**定数を可能な限り排除し、すべてを数式で表現する**

これにより、スキルやアイテムによるゲームルールの動的変更が可能になる。

**適用例**

| パラメータ | 数式アプローチ |
|-----------|---------------|
| 侵食インターバル | f(経過時間) + 調整パラメータで短縮 |
| 侵食セル数 | g(ウェーブ数) + 調整パラメータ で増加 |
| 地雷配置率 | h(領域サイズ) |
| リスポーン時間 | i(現在レベル(正の相関), スキル補正) |
| レベルアップ必要EXP | j(現在レベル) |
