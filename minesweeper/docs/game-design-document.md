## エグゼクティブサマリー

本作は、大学新歓イベント向けに開発される協力型ローグライトマインスイーパーパーティーゲームである。従来のマインスイーパーに「荒野システム」という独自の爆発連鎖機構を組み合わせ、時間経過とともに侵食される領域を仲間と守り抜く10〜15分のセッションベースの体験を提供する。最大の特徴は、すべてのパラメータを数式駆動で設計することで、スキルやアイテムによるゲームルールの動的変更を可能にし、少人数短期開発でも拡張性の高い基盤を構築できる点にある。プレイヤーは各自のデバイスでブラウザアクセスし、リアルタイムで協力しながら、レベルアップによるスキル獲得と買い物を繰り返し、最終的に15分間の生存を目指す。

# ゲームシステム

### コアメカニクス

本作は古典的マインスイーパーをベースとし、以下の独自要素を追加する。

#### 基本ルール

* プレイヤーは盤面のセルを開く  
* 数字は周囲8マスの地雷数を示す  
* 地雷を踏むと爆発が発生  
* 和音機能 (chord) は常に有効 (数字の周囲がすべてフラグ済みなら一括オープン)  
* 誤フラグペナルティなし

#### プレイヤー衝突

* 1セルの当たり判定を持つ  
* 同一セルに複数プレイヤーは同時占有不可  
* 実質的に同一セル入力競合は発生しないが対策は必要

### 荒野システム — コアディファレンシエーター

本作の最大の特徴は、地雷爆発によって生成される「荒野 (wasteland)」と、その連鎖反応である。

#### 爆発メカニクス

* **地雷起爆**: プレイヤーが地雷セルを踏むと爆発が発生  
* **爆発範囲**: 半径1セル (周囲8マス) がすべて荒野化  
* **連鎖起爆**: 爆発範囲内に別の地雷が含まれていた場合、その地雷も起爆  
* **連鎖速度**: 1セルあたり1/8秒。プレイヤーは反射神経で回避可能  
* **即死判定**: 爆発範囲内のプレイヤーは即座に死亡

#### 荒野の性質

* **保証された地雷原化**: 次回の侵食インターバルで必ず地雷原エリアに変換される  
* **フロンティアとしてカウント**: 地雷エリアの拡張は荒野の境界からも開始される  
* **BFS拡張**: 通常の侵食半径は1セル。セル数が不足する場合、それを超えて幅優先で拡張

#### 侵食後の再計算

各侵食フェーズ後、隣接するすべての数字が再計算される。プレイヤーは常に正確な情報を得られる。

```
[爆発連鎖フロー]
プレイヤーが地雷を踏む
↓
半径1セル爆発
↓
荒野化 (8セル)
↓
爆発範囲内に地雷あり？
Yes → 1/8秒後に連鎖起爆
No → 終了
↓
プレイヤーは逃走可能
↓
次回侵食で荒野は地雷原化確定
```

### 侵食システム

時間経過とともに、未開封セルが地雷エリアに変換される。

#### 侵食トリガー

* **インターバル**: 数式ベースで決定され、時間経過とともに短縮  
* **侵食セル数**: インターバルごとの侵食量も数式で増加  
* **警告時間**: 3〜5秒の視覚的予告 (色変化 \+ アニメーション)

#### 侵食メカニクス

* 1\. フロンティア隣接の未開封セルに警告マーカー表示  
* 2\. 警告後、対象セルが地雷原に変換 (地雷配置率は数式決定)  
* 3\. 荒野セルは100%地雷原化  
* 4\. 荒野エッジからBFS拡張  
* 5\. すべての数字を再計算

#### 自然なダウンタイム

長いインターバル \= 計画・買い物タイム。数式で管理され、スキルやアイテムで制御可能。

### 勝利/敗北条件

* **勝利**: 15分間生存 → 強制ドラマチックイベント → "Wave Completed\!" 表示  
* **敗北**: 全プレイヤー同時死亡 (安全領域がゼロになり侵食で全滅する自然な流れ)

### 死亡とリスポーン

#### 死亡時

* 爆発に巻き込まれる / 地雷原の復活に巻き込まれると死亡  
* 所持アイテムすべてロスト  
* リスポーンタイマー開始 (数式ベース、定数でない)

#### 死亡中の状態 (LoL方式)

* 観戦モード: 他プレイヤーの様子を見られる  
* ショッピング可能: 死亡中でも買い物ができる

#### リスポーンタイマー短縮

スキルやアイテムでリスポーン時間を削減可能。

### ボードと領域

* **ボード規模**: かなり大きく、プレイヤー数に応じてスケール  
* **スポーン地点**: 複数存在。1グループ最大4人まで (5=2+3, 6=3+3, 7=3+4, 9=3+3+3, 10=3+3+4)  
* **領域**: 開いた安全セル。侵食はフロンティアエッジを狙う

# プログレッションシステム

本作は通貨 \+ レベルアップ時ドラフト形式のローグライト進行を実装する。

| 報酬 | 獲得方法 | 用途 | 特性 |
| :---- | :---- | :---- | :---- |
| **EXP (経験値)** | 行動報酬 | レベルアップ | 累積型 |
| **Money (Gold)** | レベルアップ選択肢 / 行動報酬 | アイテム購入 | 安定資産 |

### レベルアップドラフト

* 1\. EXP蓄積 → レベルアップ  
* 2\. ランダムな3つのスキル/アイテムを提示  
* 3\. プレイヤーが1つ選択  
* 4\. スキル \+ アイテムスロットが満杯の場合、選択肢はMoneyまたは回復に変化

### ショップ

* Moneyで購入  
* いつでも購入可能 (インターバル中推奨)  
* アイテムは高価  
* 緊急用途 (早期蘇生など)

### キャラクター選択

* 初期にキャラクター選択  
* キャラクターはガチャテーブルとバフ/デバフに影響  
* MVP時に実装はなし  
* 将来的に拡張可能な設計

### メタプログレッション

* **アカウント**: ユーザー名 \= アカウント (重複不可、セキュリティ考慮なし)  
* **永続データ**: アカウントレベルで保存  
* **1日通しての進行**: イベント期間中、アカウント成長が継続  
* **リトライインセンティブ**: メタ進行により再挑戦が有意義に  
* MVP時に実装は無し  
* 将来的に閣僚可能な設計

# マルチプレイヤー・協力要素

### 部屋システム

* プレイヤー数: 可変スケーリング (1〜10+人)  
* リアルタイム同時操作  
* 途中参加: いつでも完全サポート

### リソース管理

* **完全個人制**: EXP、Money、アイテム、スキルはすべて個人所有  
* 他プレイヤーとのリソース共有なし  
* 協力は行動とタイミングで表現

### スポーンと合流

* 複数スポーン地点  
* プレイヤーは初期配置を選択できない(システム決定)  
* 合流戦略または分散戦略を自由に選択

# UI/UX

### プレイヤー表示 (Inversusスタイル)

* **形状**: 角丸四角形  
* **方向表示**: 内部に方向を示す円弧  
* **名前**: 上部にフロート表示  
* **色**: キャラクター識別色

### 警告ビジュアル

* 侵食対象セルに色変化 \+ アニメーション  
* 3〜5秒の猶予  
* 視覚的に直感的であること (チュートリアル不要)

### リザルト画面

* チーム統計  
* 個人貢献スコア  
* 再挑戦インセンティブの提示

### 視覚直感性の原則

**マインスイーパールールの口頭説明のみで理解可能なUI設計**  
チュートリアルなしでプレイ可能。すべての情報は視覚的に明瞭に伝える。

| レイヤー | 技術 |
| :---- | :---- |
| **フロントエンド** | React, TypeScript, Vite |

| **データベース** | SQLite3 (メタ進行、ハイスコア) |
| **認証** | ゲストユーザー名のみ (ユーザー名 \= アカウント) |
| **アーキテクチャ** | Authoritative Server |

### システム構成図

```
[クライアント (Browser)]
↕ WebSocket
[Colyseus Server]
↓
[Game State Manager]
├─ Board State (侵食、荒野、地雷)
├─ Player State (位置、HP、スキル、アイテム)
├─ Invasion Timer (数式駆動)
└─ Event Manager (ランダムイベント)
↓
[SQLite3 DB]
├─ Account Meta Progression
└─ High Scores
```

### 数式駆動設計の原則

**定数を可能な限り排除し、すべてを数式で表現する**

#### 適用例

| パラメータ | 数式アプローチ |
| :---- | :---- |
| 侵食インターバル | f(経過時間) \+ 調整パラメータで短縮 |
| 侵食セル数 | g(ウェーブ数) \+ 調整パラメータ で増加 |
| 地雷配置率 | h(領域サイズ) |
| リスポーン時間 | i(現在レベル(正の相関), スキル補正) |
| レベルアップ必要EXP | j(現在レベル) |
