# Detonator 設計レビュー vol.3-1 (第二ラウンド)

**Date:** 2026-03-24

**Note:** 前回36件はすべて解決済み。本ファイルは新規発見のみ。

---

1. **[HIGH] Hole通行不可確定後、CP配置制約が不足しておりクリア不能フロアが発生しうる**
   - **根拠**:
     - 回答 vol.3 C2: Holeは通行不可で確定
     - B5-42: CPはセル種類と独立（座標オブジェクト）
     - B3-24/B5-43: 到達不能防止不要という前提
   - **新規性**: 以前はHole通行可否自体が未確定だったが、vol.3で「不可」に確定したことで、CPがHole座標に置かれた場合の詰み条件が新たに顕在化。
   - **未確定点**: CP候補からHoleを除外するのか、あるいはHole上CPを自動移設するのかが未定義。

2. **[HIGH] 「同tick順不同（ランダム）」×「同tick爆発は段階適用」の組み合わせで結果再現性が崩れる**
   - **根拠**:
     - 回答 vol.3 C3: 同tick内の操作順はランダム
     - 回答 vol.3 C4: 同tick内でも順次処理し前の爆発結果を後段に反映
     - A4設計注意: Primのタイブレークは決定的にすべき
   - **新規性**: 旧レビューの「未定義」は解消済みだが、vol.3回答同士の組み合わせで“仕様としてランダム依存”が導入され、同条件の再現テスト/検証が不安定になる。
   - **未確定点**: ランダム順を許容するなら、乱数シード固定規則（tick seed, room seed）とログ再現規約が必要。

3. **[HIGH] 踏み起爆BFS（vol.3 M16）における「爆発後セル遷移」の共有契約が未定義**
   - **根拠**:
     - 回答 vol.3 M16: 踏み起爆はDetonateとは別パイプライン（BFS連鎖）
     - A3-11: 衝撃波直径3 / 爆風直径5 の距離定義はある
     - B4-32/33: 「セル変化で旗除去」は定義されるが、踏み起爆系へ同一適用する明示がない
   - **新規性**: 旧レビューでは「別系統かどうか」が論点だったが、確定後に「別系統でも後処理契約は共通か」が新たな欠落として残った。
   - **未確定点**: 踏み起爆でも Detonate と同じく
     1) 荒地化範囲適用、
     2) 旗除去、
     3) Relay Point/安全地雷ノードへの影響
     を同一ルールで処理するか未確定。

4. **[MEDIUM] 複数人同一セル許容下で、侵食変換時の同時死亡裁定が未明文化**
   - **根拠**:
     - B2-18 + vol.3 H2: 同一セル内に複数人存在しうる
     - A5-18: 侵食対象セル上のプレイヤーは即死
   - **新規性**: 「1人が立っている場合」の定義はあるが、「同一セル複数人時に全員同時処理か、順次処理か」の明文化がない。
   - **未確定点**: 全員同時即死なら、Nine Lives/Disposable Life発動順・全滅判定時刻を同tickでどう確定するかが必要。

5. **[MEDIUM] Dashと荒野減速の合成則が未定義（速度計算の実装分岐が残る）**
   - **根拠**:
     - B6-47の例でDashアイテム存在が示唆
     - B2-12: 荒野は基礎速度×0.4
     - vol.3でDash効果式（倍率/秒数/合成順）の規定なし
   - **新規性**: 旧36件には含まれていないが、vol.3時点でも移動式の最終形が未確定。
   - **未確定点**: Dash中の荒野速度を
     - `base * dash * 0.4` とするのか、
     - `wastelandSpeed * dash` とするのか、
     - あるいは荒野減速を無視するのか
     を固定する必要がある。

6. **[MEDIUM] Mine Remover（方向除去）と遅延爆発キュー（1/8秒連鎖）の干渉ルールが未定義**
   - **根拠**:
     - B6-53 + vol.3 M8: Mine RemoverはFacing依存で方向除去
     - A1-5/B4-36 + vol.3 M16: 爆発連鎖は1/8秒遅延
   - **新規性**: 旧レビューではFacing量子化が主論点。vol.3で量子化は解決したが、キュー済み爆発対象を除去した場合のキャンセル/不発判定が新たに未定義。
   - **未確定点**: 「点火済みセルを除去したとき、爆発イベントを取消すか」「既にキュー投入済みの子連鎖を残すか」の仕様固定が必要。
