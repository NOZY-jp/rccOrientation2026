## Part A: 論理的穴・矛盾・クリティカル問題

> 以下は仕様内の矛盾や、実装前に解決しなければならない設計上の欠落をまとめたものです。

### A1. 旧GDDとの重大な矛盾 [CRITICAL]

1. **勝利条件の二重定義**
   - 旧GDD: 「15分間生存 = 勝利」
   - 新仕様: 「10フロアクリア = 勝利」
   - どちらを採用するか、または両方をどう組み合わせるかが未定。

2. **経済システムの有無**
   - 旧GDD: Money（通貨）+ Shop（購入）システムあり
   - 新仕様: 経済システムへの言及が一切ない
   - Shopなしの場合、アイテム獲得経路は「ドロップ + レベルアップ提示」のみになるか？

3. **プレイヤー人数の未定義**
   - 旧GDD: 1〜10+人（可変スケーリング）
   - 新仕様: 言及なし
   - プレイヤー人数は盤面サイズ、地雷密度、チェックポイント数、起爆CT等に影響するため必須。

4. **和音（chord）機能の有無**
   - 旧GDD: 「数字の周囲がすべてフラグ済みなら一括オープン」を常時有効
   - 新仕様: 言及なし
   - これの有無がゲームペースに大きく影響する。

5. **爆発連鎖速度**
   - 旧GDD: 1セルあたり1/8秒の遅延で連鎖
   - 新仕様: タイミング未定義
   - プレイヤーが「逃げる」設計なら遅延が必要だが、即時連鎖なら異なるバランスになる。

### A2. 仕様内の論理的欠落 [CRITICAL]

6. **全員同時死亡時の処理が未定義**
   - 死亡条件: 爆風、侵食飲み込み、ランダムイベント
   - リスポーン短縮には「生存プレイヤーがアイテムを発見する」ことが必須
   - 全員死亡時、生存者がいない → リスポーン不可能 → ソフトロック？
   - 即ゲームオーバーか、救済カウントダウンがあるかが未定義。

17. **チェックポイントが到達不能になる可能性** — **問題なし**。CPはセル属性でありセル種類とは独立。侵食されても消滅せず座標に残る。プレイヤーがその座標に触れれば獲得可能。経路断絶の概念も不適切。

8. **スポーン地点の侵食**
   - 侵食は「前線」から内側に向かって進むが、初期安全ゾーンが削り尽くされた場合、スポーン地点自体が侵食されるか？
   - スポーンが侵食されたらリスポーン先が消失する。

9. **死亡時アイテム喪失の対象範囲**
   - 「アイテムをすべて失う」とあるが、スキルは失わない前提か？
   - Nine Lives（自動発動型蘇生アイテム）は死亡時に発動するが、死亡＝全アイテムロストなら九死に一生も失われるのか？

10. **「向き」の定義がない**
    - 地雷除去機（安い/普通/高い）が「プレイヤーの向いている方向」を参照する。
    - グリッドベースゲームで「向き」をどう定義するか？（移動方向準拠/最終入力方向固定/独立エイム）
    - これが未定義だと方向性アイテムが実装できない。

### A3. 距離定義の曖昧さ [HIGH]

11. **「直径3マス」「直径5マス」の距離定義**
    - マンハッタン距離かチェビシェフ距離かで範囲が大きく異なる。
    - チェビシェフ距離（斜め含む）なら「直径3」= 7マス、マンハッタンなら最大5マス。
    - 侵食警告、起爆探索、チェックポイント検知、爆風範囲すべてに影響する。

12. **「直径」の解釈**
    - 「直径3」は「中心+周囲1マス」を意味するか（チェビシェフ半径1）、それとも「中心から3マス先まで」を意味するか（半径3）？

### A4. 起爆システムの未定義 [HIGH]

13. **貪欲な最短経路のタイブレーク**
    - 複数の旗への最短経路が同長の場合、どの経路を選ぶか？
    - ランダム？固定優先順位（上>右>下>左）？

14. **経路上の障害物**
    - 起爆の導火線は穴や荒野を通過できるか？
    - 通過不可なら、盤面レイアウトによっては起爆がほぼ機能しない。

15. **旗が非地雷セルにある場合の挙動**
    - 「地雷でない場所に旗を立てると誘爆がそこで停止」とあるが、そもそも安全マスに旗を立てることは可能か？
    - 可能な場合、誤爆のリスクが常にあることになる。

16. **複数プレイヤー同時起爆**
    - 同フレームで2人が起爆した場合、サーバー側でどう処理するか？
    - 順次処理？同時適用？

### A5. 侵食のエッジケース [HIGH]

17. **侵食対象セル上のアイテム**
    - 侵食によってセルが変化した場合、その上にあるドロップアイテムはどうなる？
    - 消滅？残置？プレイヤーに飛ぶ？

18. **侵食対象セル上のプレイヤー**
    - 警告アニメーション中表示されているセルにプレイヤーが立ち続けた場合、侵食発動時にどうなる？
    - 即死？（警告の意味がない）
    - 侵食発動直前にノックバック？

19. **侵食で安全マスが地雷原に変わった場合の数字再計算**
    - 新仕様では「数字がないマスは一気に掘られる」とある。
    - 侵食で周囲の地雷状況が変わった場合、既存の安全マスの数字は再計算されるか？
    - 再計算されないと、数字が不正確になりゲームが成立しない。
    - （旧GDDでは「各侵食フェーズ後、隣接するすべての数字が再計算される」と明記あり）

20. **侵食警告中にDigされた場合**
    - 警告中のセルをプレイヤーがDigで安全マスにした場合、侵食はキャンセルされるか？

## 回答

> 以下は設計者による回答。回答日: 2026-03-24

### A1. 旧GDDとの重大な矛盾

1. **勝利条件**: 10フロアクリアのみ採用。15分生存制は廃止。
2. **経済システム**: 完全撤去。Money/Shopなし。アイテム獲得はドロップ＋レベルアップ提示のみ。
3. **プレイヤー人数**: 1~10+人（旧GDD維持）。可変スケーリング。
4. **和音（chord）機能**: 常時有効ではなく、一回きり取得できるレアスキルの一つとして実装する。
5. **爆発連鎖速度**: 1/8秒/セルを維持（旧GDD通り）。

### A2. 仕様内の論理的欠落

6. **全員同時死亡時**: 即ゲームオーバー。救済措置なし。
9. **九死に一生と死亡時アイテムロスト**: 九死に一生は「死を回避する」アイテムであり、発動した時点で死んでいない扱いとなるため、アイテムロストのペナルティは発生しない。
10. **「向き」の定義**: 最終移動方向固定。最後に移動した方向がプレイヤーの向きになる。

### A3. 距離定義

11. **距離定義は場面ごとに別々に設定**:
    - 誘爆範囲（衝撃波）直径3: チェビシェフ距離、半径1（周囲8マス）
    - 荒地化範囲（爆風）直径5: マンハッタン距離、半径2（縦横2マス、斜め1マス）
    - チェックポイント検知: **ユークリッド距離**。プレイヤー位置は連続座標なので `dist² ≤ R²` で計算。
    - 起爆探索（導火線リーチ）: **チェビシェフ距離**（旗と旗の間の距離）
    - 侵食警告: **距離概念ではない**。次の侵食フェーズで地雷原になるマスが直接的に警告対象。frontlineに隣接するマス = 警告対象。

### A4. 起爆システム

13. **起爆経路アルゴリズム** → **Rooted Prim Tree + 安全地雷枝刈り**方式:
    - MSTアルゴリズムは **Prim** 採用（Kruskal不採用）。理由: 小さな連結成分を個別に処理する用途に最適。旗を置くたびに全エッジをソートするKruskalは不適切。
    - 連結成分（BFS/DFSで範囲内旗を探索）は**候補プール**に過ぎず、爆発論理の権威ではない。
    - 権威は **Rooted Prim Tree**: 起爆源を根としてPrimでMSTを構築し、根からBFS/キューで伝播。
    - **伝播ルール**: 危険地雷に到達 → 爆発 + 子ノードをキューに追加。安全地雷に到達 → 爆発 + その枝は終了（子はキューに追加しない）。
    - 各枝は独立。1つの枝が安全地雷で止まっても他の枝は継続。
    - 安全地雷を点火源にした場合 → 爆発するが連鎖なし（実質不発）。

14. **経路上の障害物** → 穴と荒野を導火線が通過可能。

15. **旗が安全な地雷原にある場合** → 設計上許容。MST上では導火線が繋がって見えるが、伝播はそのノードで物理的に切れる。

    - 点火 → 3.0秒遅延 → 爆発タイミングで盤面スナップショット取得 + MST再計算
    - 同一tick内のイベントは1バッチとして1スナップショットに対して評価（順序バイアスなし）
    - 起爆源が既に消費済みの保留イベントはキャンセル
    - Aの爆発がBの安全地雷（伝播停止ノード）を吹き飛ばした場合 → Bの再計算時に伝播が貫通可能になる
    - 3.0秒のfuse中に旗が追加/除去されても、爆発時スナップショットのみが権典（中間変化は無視）
    - クライアントには点火時に「予測/仮」パスを表示し、「provisional」であることを明示

**設計上の注意点**:
- Primのタイブレークは**決定的**に（ランダムだとプレイヤーが予測不可能）
- 距離R付近の境界判定は `dist² ≤ R²` で整数演算し安定化
- 空間バケツインデックス（cell size ≈ 起爆範囲R）でO(n²)を回避しほぼ線形時間で旗の探索を行う

## 回答 vol.4（第二ラウンドレビューへの対応）

> 以下は interview3-1.md / interview3-2.md の新規12件に対する回答。回答日: 2026-03-24

### 用語整理

- **Detonate（点火）**: プレイヤーコマンド。MSTで旗付き地雷を安全マスに変える。旗と経路上のRelay Pointは吹き飛ぶ。衝撃波なし、荒地化なし。
- **管理外爆発（起爆）**: 危険地雷を誤って掘った場合。衝撃波(チェビシェフ半径1)+荒地化(マンハッタン半径2)+旗/Relay Point除去。
- 二つのシステムは別パイプライン。共通点: 効果範囲内の旗とRelay Pointは吹き飛ぶ。

### HIGH

**3-1#1. CP on Hole** → **CP候補からHoleを除外**。CPは歩ける場所にのみ配置。

**3-1#2. 同tick再現性** → **ランダム順を許容**。実装によって適当に決まった順番で処理される。テスト時はseed固定で対応。

**3-1#3/3-2#1. 爆発プロファイル共通化** → **Detonate（点火）と管理外爆発（起爆）は別物**。共通点は「効果範囲内の旗とRelay Pointは吹き飛ぶ」のみ。起爆のみ衝撃波・荒地化・致死が発生。

**3-2#2. tick内イベント優先順位** → **固定順序を定義**（実装時に詳細化）。基本: 入力確定→期限到来イベント処理→爆発適用→再計算。

**3-2#5. フロア遷移時CT/バフ** → **CT・バフ・一時効果はリセット**。持ち越すのはアイテム・スキルのみ。

**3-2#6. 非playing中disconnect** → **特別な処理不要**。切断していたプレイヤーは再接続時にその時点のゲーム状態から復帰。フロア遷移中なら次フロアの最初から参加できる。

**3-2#8. mid-movement侵食判定** → **tick開始位置で判定**。tick終了位置は考慮しない。

### MEDIUM

**3-1#4. 複数人侵食即死** → **全員同時即死**。

**3-1#5/3-2#9. 速度合成** → **乗算合成**。`base × wasteland(0.4) × dash(1.5) × (1 + skillStack)`。

**3-1#6. Remover×キュー済み爆発** → **キャンセル**。除去した時点でキュー済みの爆発イベントもキャンセルされる。

**3-2#3. Relay Point耐性** → **吹き飛ぶ**。侵食・管理外爆発・Detonateいずれでも吹き飛ぶ。

**3-2#4. 爆発と地上ドロップ** → **消滅**。管理外爆発（衝撃波+荒地化）に巻き込まれた地上ドロップは消滅。

**3-2#7. level-up中途中参加** → **「level-upフェーズ」は存在しない**。レベルアップしたら報酬が溜まっていくだけ。フロアクリア後の休憩時間にまとめて選択。途中参加者が休憩時間中に参加したらその時点で選択可能。
