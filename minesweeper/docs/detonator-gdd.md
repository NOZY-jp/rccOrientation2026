# Detonator — ゲームデザイン文書

## 1. エグゼクティブサマリー

Detonatorは、大学新歓イベント向けに開発される協力型ローグライトマインスイーパーパーティーゲームである。従来のマインスイーパーに「旗起爆システム」という独自の制御連鎖爆発メカニクスと、「侵食システム」という時間経過に伴う盤面縮小メカニクスを組み合わせ、地雷原の中に隠されたチェックポイントを探し出し回収する10フロア構成の体験を提供する。

最大の特徴は、すべてのパラメータを数式駆動で設計することで、スキルやアイテムによるゲームルールの動的変更を可能にし、短期的なハックソン開発でも拡張性の高い基盤を構築できる点にある。プレイヤーは各自のスマートフォンからブラウザアクセスし、仮想ジョイスティックと仮想ボタンで操作してリアルタイムで協力しながら、レベルアップによるスキル獲得とアイテム収集を繰り返し、最終的に10フロアの攻略を目指す。

## 3. セルタイプ (Cell Types)

盤面を構成するセルは以下の5種類である。

| セルタイプ | 移動 | 掘削 | 説明 |
|------------|------|------|------|
| 安全マス (Safe) | 可能 | 不可 | 掘られたマス。周囲の地雷数を数字で表示。初期スポーン地点。 |
| 安全な地雷原 (Safe Mine) | 不可 | 可能 → 安全マス | 地雷なし。危険な地雷原と目視で見分け不可。 |
| 危険な地雷原 (Dangerous Mine) | 不可 | 可能 → 爆発 | 地雷あり。安全な地雷原と目視で見分け不可。 |
| 荒地 (Wasteland) | 可能 (速度低下) | 不可 | 移動速度60%低下。爆発衝撃波で発生。 |
| 穴 (Hole) | 不可 | 不可 | 初期配置。ステージ効果で増殖する可能性あり。橋アイテムで安全マスに変化。 |

### 目視不可の地雷原

安全な地雷原と危険な地雷原は**視覚的に区別できない**。プレイヤーは隣接する安全マスの数字を観察することで、地雷の有無を推察する。これは通常のマインスイーパーと同様の推論メカニクスである。

## 4. プレイヤー操作

### 移動

- 安全マスと荒地のみ移動可能
- 地雷原と穴はコリジョンがあり進めない
- 1セルずつ8方向の移動 (チェスのキングと同様)
- 同一セルに複数プレイヤーの同時占有は不可 (1プレイヤー/セル)
- プレイヤーの向き: 最後に移動した方向で決定 (地雷除去機アイテムで参照)

### 基本アクション (無制限)

| アクション | リーチ | 説明 |
|-----------|--------|------|
| 旗を立てる | 1マス (8方向) | 対象セルに旗を設置。マーカーとして機能。 |
| 掘る | 1マス (8方向) | セルを掘削。結果は対象セルの種類による。 |

**掘る結果**:
- 安全な地雷原 → 安全マスに変化。数字なしマスの場合は隣接する数字なしマスも一括掘削 (flood-fill)。安全マスを作った行為にEXP発生。
- 危険な地雷原 → 管理されていない爆発が発生 (セクション6参照)。

### 特殊アクション

| アクション | 対象 | クールダウン | 説明 |
|-----------|------|-------------|------|
| 起爆 | 旗が立っているセル | デフォルト10秒 (プレイヤー人数で変動) | 管理された連鎖爆発。セクション7参照。 |

## 5. チェックポイントシステム

### 概要

各フロアにはステージごとに決まった数のチェックポイントが配置されている。すべてのチェックポイントを回収することがフロアのクリア条件である。

### 配置

- チェックポイントは**地雷原の中に最初から存在する**
- ステージごとの候補地の中からランダムに配置
- 候補地と配置数はステージ定義に含まれる

### 検知

- チェックポイントは**通常不可視**
- プレイヤーが検知範囲 (直径3マス) に近づくと可視化される
- 検知範囲はスキル/アイテム/ステージ効果によって変化

### 回収

- チェックポイントのセルを掘って安全マスにする、または到達することで回収
- 全チェックポイント回収でフロアクリア

### 侵食との関係

チェックポイントは元々地雷原の中にあるため、侵食によって「飲み込まれる」という概念はない。侵食対象になったとしても、それは元から地雷原であった場所である。

## 6. 管理されていない爆発 (Uncontrolled Explosion)

トリガー: プレイヤーが危険な地雷原を掘る。

```
[管理されていない爆発フロー]

危険な地雷原を掘る
        ↓
  ┌─── Phase 1: 爆発 (直径3マス = 中心 + 周囲8マス) ───┐
  │                                                       │
  │  範囲内のセル:                                        │
  │  ├─ 地雷あり → 連鎖起爆 (別セルで再帰的に Phase 1+2)  │
  │  ├─ 安全マス → 荒地化                                 │
  │  ├─ 地雷原 → 荒地化                                   │
  │  └─ プレイヤー接触 → 死亡                             │
  │                                                       │
  └───────────────────────────────────────────────────────┘
        ↓
  ┌─── Phase 2: 衝撃波 (直径5マス、同心円・中心同じ) ────┐
  │                                                       │
  │  直径3〜5の範囲のセル:                                │
  │  ├─ 安全マス → 荒地化                                 │
  │  ├─ 地雷原 → 荒地化                                   │
  │  ├─ 連鎖起爆はしない (Phase 1のみ連鎖)               │
  │  └─ プレイヤー接触 → 死亡                             │
  │                                                       │
  └───────────────────────────────────────────────────────┘
        ↓
  連鎖速度: 1セルあたり1/8秒 (プレイヤーは反射神経で回避可能)
```

### 重要事項

- **Phase 1**が地雷を巻き込むと、その地雷を中心に再帰的に爆発が発生する
- **Phase 2 (衝撃波)** は連鎖起爆を起こさない。荒地化のみ
- 両フェーズともプレイヤーが範囲内にいると死亡
- 爆発によって地形が大きく変化するため、慎重な推論が求められる

## 7. 管理された爆発 — 起爆コマンド (Controlled Explosion)

トリガー: 旗が立っているセルに対して起爆コマンドを使用。

```
[起爆コマンドフロー]

旗が立っているセルで起爆コマンド使用
        ↓
旗の下に雷管+導火線を設置 (概念的)
        ↓
貪欲な最短経路で探索 (直径5マス以内)
        ↓
他の旗の下の地雷に到達？
    ├─ Yes → 管理された誘爆発生
    │         ├─ 地形に影響なし (荒地化しない)
    │         ├─ プレイヤーにダメージなし (死なない)
    │         ├─ コンボ数に応じてEXP獲得量増加
    │         └─ 連鎖深度は無限 (旗B→旗C→旗D→...)
    │
    └─ No → 何も起こらない
        ↓
地雷ではない場所に旗があった場合:
    ├─ 連鎖がその旗で停止
    └─ プレイヤーはペナルティクールダウンに入る [要検討: 具体的秒数]
```

### クールダウン

| 状態 | クールダウン | 備考 |
|------|-------------|------|
| 正常起爆 | デフォルト10秒 | プレイヤー人数で変動 [要検討: 具体的な式] |
| ペナルティ (地雷なし旗で連鎖停止) | 通常より長い | [要検討] |

### 戦略的意味

- 旗は正確な推論に基づいて設置する必要がある
- 誤った場所に旗を立てて起爆すると連鎖が停止し、長いペナルティを受ける
- 高いコンボ数は大量のEXPを獲得できるため、旗の配置パズルがゲームの主要な戦略要素となる
- リレーポイントアイテムを利用して距離を稼ぐ中継点戦略が可能

## 8. 侵食システム (Erosion System)

時間経過とともに、未掘削セルが地雷原に変換される。

```
[侵食フロー]

侵食インターバル経過 (式で決定)
        ↓
侵食力を算出 (式で決定 = 今回変換するマス数)
        ↓
前線からランダムに1マスを選択 (起点)
        ↓
起点から幅優先探索 (侵食力分のマスを消費)
    ├─ 前線だけで足りる → 前線のみ探索
    └─ 足りない → 前線より1マス内側も探索
        ↓
対象マスに警告アニメーション表示
        ↓
警告時間経過 (侵食インターバルに依存)
        ↓
対象マスがランダムな種類の地雷原に変化 (安全 or 危険)
```

### 前線の定義

前線とは、安全マスとそれ以外 (地雷原/荒地/穴) の境界から1マス内側にある安全マスのこと。実質的には数字が表示されているマスの最外周。荒地が隣接している安全マスも前線に含まれる。

### 侵食パラメーター

すべて数式で決定される。パラメーターの一覧はセクション14を参照。

### 警告時間の設計思想

侵食インターバルが短い段階では、警告時間も短くする。インターバルが短いのに長い警告を表示すると、盤面が警告だらけになり情報過多になるため、UXを考慮して警告時間をインターバルに連動させる。具体的な式は [要検討]。

## 9. レベルアップとローグライト進行

### EXPシステム

| 行動 | EXP |
|------|-----|
| 安全な地雷原を掘って安全マスを作る | 発生 [要検討: 具体的量] |
| 数字なしマスがflood-fillで一気に開く | 各マスで発生 |
| 起爆コンボ | コンボ数 × 倍率 [要検討] |

レベルアップに必要なEXPは指数関数的に増加する: `requiredEXP = i(currentLevel)`

### レベルアップドラフト

1. EXP蓄積 → レベルアップ (即時発生)
2. ランダムな3つのスキル/アイテムから1つを選択
3. **報酬は貯めておける** — すぐに選ばなくてもよい
4. フロア移動中の休憩時間に落ち着いて選択することを想定
5. 余裕があればプレイ中に選択してもよい

### スキルシステム

- スキルスロットは**存在しない**
- スキルはパッシブに自動適用され、**無限スタック可能**
- 同じスキルを複数回取得できる (例: 移動速度+2% を5回取得 = +10%)
- 1つ1つの効果は意図的に弱く設定 — 大量に積む想定

| スキル名 | 効果 | 備考 |
|----------|------|------|
| リスポーン時間減少 | 1〜3秒 | スタック可 |
| 移動速度上昇 | 2〜6% | スタック可 |
| 点火クールダウン減少 | 0.5〜1秒 | スタック可 |
| 経験値獲得増加 | 5〜20% | スタック可 |
| コンボ倍率 | +0.05x〜+0.2x | スタック可 |
| 侵食クールダウン増加 | 5〜10% | 侵食間隔を延長 |
| アイテムドロップ率増加 | +2〜4% | スタック可 |
| アイテム吸収半径増加 | +0.2〜0.5マス | スタック可 |
| アイテムスロット増加 | +1 | スタック可 |
| チェックポイント検知範囲増加 | +0.5マス | スタック可 |
| 侵食予知 | 警告表示が0.5秒早まる | スタック可 |
| 死亡時アイテム保持確率 | +5% | スタック可 |
| 荒地移動速度ペナルティ軽減 | 2〜5% | スタック可 |

## 10. アイテムシステム

### ドロップ条件

- 安全な地雷原を掘った時、各マスで独立してドロップ抽選
- flood-fillで数字なしマスが一気に開いた時も各マスで独立抽選
- ベースドロップ率: 10% (スキル/アイテム/ステージ効果で変化)
- ドロップテーブル: 外部データファイルで管理 [要検討: JSON/YAML/TOML]
- 当選するとそのマスにアイテムが出現

### ピックアップ

- プレイヤーがアイテムに触れると、スロットに空きがあれば自動で格納
- 吸収半径: ベース半径 + スキル補正

### アイテムスロット

- ベース: 3個 (スキルで増加可能)
- タップで説明表示 → 使用/捨てるを選択
- スロット満杯時の挙動: [要検討]

### 発動方式

アイテムによって異なる:
- **手動使用**: スロットから選択して発動 (大部分のアイテム)
- **自動発動**: 条件を満たすと自動使用 (九死に一生等)

### アイテム一覧

| アイテム名 | 効果 | タイプ |
|-----------|------|------|
| リレーポイント | 安全マスに旗を立てる。距離が足りない時のコンボ中継点。 | 手動使用 |
| ダッシュ | 15秒間移動速度1.5倍。 | 手動使用 |
| 強制点火 | クールダウンを無視して起爆コマンドを実行。 | 手動使用 |
| 安い地雷除去機 | 向いている地雷原1マスを安全マスにする。 | 手動使用 |
| 普通の地雷除去機 | 向いている地雷原を、奥2マス×横3マスを安全マスにする。 | 手動使用 |
| 高い地雷除去機 | 向いている地雷原を、奥4マス×横6マスを安全マスにする。 | 手動使用 |
| チェックポイント探知 | 最寄りのチェックポイントを5秒間可視化。 | 手動使用 |
| (名前未定) | 自分が立っている場所を中心に9マスの荒地を安全マスに戻す。 | 手動使用 |
| 緊急脱出 | リスポーン地点にテレポート。 | 手動使用 |
| 小休憩 | 侵食を5秒間完全に停止。 | 手動使用 |
| 休憩 | 侵食を15秒間完全に停止。 | 手動使用 |
| 橋掛け | 穴を安全マスに変化させる。 | 手動使用 |
| 使い捨て命 | 使用から10秒以内に死亡するとノーコストでリスポーン。 | 手動使用 |
| 九死に一生 | 自動使用。死亡時1回ノーコストでリスポーン。 | 自動発動 |

### 復活アイテム (特殊ドロップルール)

- 誰かが死亡している場合: アイテム出現テーブルの**90%** (ステージ効果で変化) が復活アイテムに変化
- 復活アイテム効果: リスポーン残り10秒以上なら半減、未満なら-5秒
- 誰も死亡していない場合: 復活アイテムの出現率 **0%**
- **事実上、生存プレイヤーの助けがないと復帰は困難な設計**

## 11. 死亡とリスポーン

### 死亡条件

1. 管理されていない爆発の爆発範囲 (直径3マス) に触れる
2. 管理されていない爆発の衝撃波範囲 (直径5マス) に触れる
3. 侵食が発生した際、警告アニメーションが表示されていたマスから退避できなかった
4. ランダムイベントに巻き込まれた [要検討: 具体的イベント内容]

### 死亡時のペナルティ

- 所持アイテムを**すべてロスト**
  - スキル「死亡時アイテム保持確率+5%」で一部保持の可能性あり (各スタックごとに独立判定を想定)
- リスポーンタイマー開始
  - ベース: 40秒
  - プレイヤーのスキルで計算 [要検討: 具体的な式]

### リスポーン

- **場所**: 生存プレイヤーの周囲の地雷原でないマス
- 全プレイヤー同時死亡 → ゲームオーバー (敗北)
- 復帰には生存プレイヤーの協力が実質必須 (復活アイテムは生存者が掘らないと出現しないため)

## 12. フロア構造

### フロア数と勝利条件

- **10フロアクリア**で勝利
- 後で追加するかもしれないし、面白ければ無限城にしてもよいが現時点では構想段階

### ステージ選択

- 各フロアはフロアごとに用意されたステージプールからランダムに決定
- ステージ定義: 盤面レイアウト、地雷配置率、チェックポイント候補地、穴の配置、ステージ効果

### フロアの流れ

```
[フロアフロー]

フロア開始
    ↓
プレイヤーがスポーン地点に配置
    ├─ 複数スポーン地点、システム決定
    └─ 1グループ最大4人 (例: 5人=2+3, 10人=3+3+4)
    ↓
チェックポイントは地雷原内に隠蔽配置済み
    ↓
プレイヤーが探索・掘削・旗設置・起爆・チェックポイント回収
    │
    ├─ 並行: 侵食が進行、安全領域が縮小
    ├─ 並行: レベルアップ報酬の選択 (貯めておいて休憩時に選択推奨)
    └─ 並行: アイテム収集・使用
    ↓
全チェックポイント回収 → フロアクリア
    ↓
フロア移動 (休憩時間)
    ├─ レベル・スキル・アイテムをすべて引き継ぐ
    └─ 貯めていたレベルアップ報酬を選択
    ↓
次フロア開始
```

### ステージ効果

[要検討] 各ステージが固有の修飾子を持つ可能性。例:
- 侵食速度の変化
- 地雷比率の変化
- 特殊地形の出現
- ステージ効果は数式のパラメーターを変更する形で適用

## 14. 数式駆動設計

### 原則

**定数を可能な限り排除し、すべてを数式で表現する。** これにより、スキルやアイテムによるゲームルールの動的変更を可能にする。

### パラメーター一覧

| パラメータ | 式のパラメーター | 備考 |

### [要検討] 具体的な式の定義

## 16. UI/UX設計方針

### 設計原則

**「マインスイーパールールの口頭説明のみで理解可能なUI設計」**

チュートリアルなしでプレイ可能。すべての情報は視覚的に明瞭に伝える。

### プレイヤー表示

- Inversusスタイルの角丸四角形
- 内部に方向を示す円弧 (最後に移動した方向)
- 上部に名前をフロート表示
- キャラクター識別色

### 侵食警告

- 対象セルに色変化 + アニメーション
- 警告時間は視覚的に直感的に伝わること

### 情報優先度

- 盤面ファースト・最小HUD

## 17. 技術アーキテクチャ

### 技術スタック

| レイヤー | 技術 |
|----------|------|
| フロントエンド | React, TypeScript, Vite |
| バックエンド | TypeScript (Colyseus) |
| 状態管理 | Zustand |
| レンダリング | Canvas/WebGL (Phaser 3候補) |
| 認証 | ゲストユーザー名のみ (ユーザー名=アカウント) |
| アーキテクチャ | Authoritative Server |
| 物理演算 | カスタムAABB (衝突・押し合いデモ検証済み) |

### システム構成

```
[クライアント (ブラウザ/スマホ)]
    ↕ WebSocket
[Colyseus Server]
    ↓
[Game State Manager]
    ├─ Board State (セル、地雷、旗、チェックポイント)
    ├─ Player State (位置、向き、EXP、レベル、スキル、アイテム)
    ├─ Erosion Timer (数式駆動)
    └─ Event Manager (ランダムイベント)
```
